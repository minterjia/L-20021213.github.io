简单给hexo加个后台

发表于2022-01-20|更新于2022-05-30|📚学习记录
|阅读量:

今天偶然看到群里大佬谈论了这个项目，给Hexo添加一个后台，看界面挺好看的，于是就动手尝试了一下。

Qexo

项目地址

项目介绍

Qexo 是一个快速、强大、漂亮的在线 Hexo 编辑器,支持在线新建编辑文章 新建编辑单独页面 友情链接 还支持页面管理 自定义图床等功能。当然，最让我心动的就是可以在线编辑发布文章。

如何部署

申请 MongoDB

  1. 注册 MongoDB 账号 登录点击 Databases - BuildaDatabase - Shared 创建免费 MongoDB 数据库。
  2. 运营商选择AWS,地区推荐选择 N. Virginia (us-east-1) 然后点击右下角Create Cluster创建集群。
  3. 在Security Quickstart页面添加一个用户名和密码并记录，后面要用。
  4. 回到主页面侧边栏点击 Databases - Connect 连接数据库，IP地址选择Allow Access from Anywhere允许从任何位置访问，然后下一步，按照下面的图片配置将圈出的内容记下即可关闭网页。

Vercel 部署

点击按钮开始部署

在项目部署界面添加环境变量 Environment Variables

名称意义示例
DOMAINS你所允许通信的安全域名 注意双引号[“.vercel.app”, “127.0.0.1”, “.sianx.com”]
MONGODB_HOSTMongoDB 连接地址mongodb+srv://cluster0.xxxx.mongodb.net
MONGODB_PORTMongoDB 通信端口 默认应填写 2701727017
MONGODB_USERMongoDB 用户名qianxiao
MONGODB_DBMongoDB 数据库名qianxiao
MONGODB_PASSMongoDB 密码anranxxxxxxxx

点击 Deploy 开始部署，若没有 Error 信息即可打开域名进入初始化引导

添加完环境变量之后部署，部署结束之后就可以进入配置的引导页面，按照引导填入相关的内容即可，这里注意Vercel的项目ID是在项目设置的最底部项目编号。

小结

总的来说有一个后台还是会方便很多的，改错别字，发布文章什么的，随着兴趣的淡化，更新的频率也越来越低，在后台发布还不会因为网络出现什么推送失败的问题，整体来说还是很方便的。
本篇文章就是在后台直接发布的
